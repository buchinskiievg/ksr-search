# КСР — семантический поиск

Простое веб-приложение: даёшь позицию (например, из сметы) — находит наиболее похожие записи в **Классификаторе строительных ресурсов** (3 листа: Материалы, Оборудование, Машины и механизмы, ~175 тыс. строк).

Два режима:
- **Одна позиция** — ввёл строку, получил топ-N.
- **Список позиций** — пастишь список построчно, обрабатывает за один проход.

## Как это работает

**Pipeline (3 стадии):**

1. **AI-расшифровка** (опционально, Groq Llama 3.3 70B → Gemini 2.0 Flash fallback на 429)
   - Берёт ввод/аббревиатуру/жаргон или название работы («Ретрофит ячейки ввода (замена ВЭ на ... 3150 А)», «БДЛ-1», «КТП-630/10/0,4»)
   - Извлекает *предмет* из *работы* (отбрасывает «ретрофит/замена/монтаж», ищет item)
   - Расширяет аббревиатуры через ВВ-словарь (ВЭ → выключатель элегазовый, БСВ → ячейка КРУ секционная, КРУЭ, ШВЗПС, ТТ, ТН, ОПН, БМРЗ и т.д.)
   - Возвращает 2–4 варианта в канонической номенклатуре

2. **TF-IDF поиск** по нормализованным наименованиям КСР
   - word n-gram (1–2) + char_wb n-gram (3–5), score = 0.65·word + 0.35·char
   - При нескольких AI-вариантах: score = max по вариантам
   - Фильтр листа применяется до ранжирования
   - Достаёт top-20 кандидатов в первичный пул

3. **AI re-ranking** (опционально, тот же LLM-пул)
   - Получает запрос + 20 кандидатов, выставляет каждому 0–3:
     - 3 = точно, 2 = тот же класс/другие параметры, 1 = область близкая, 0 = мусор
   - Сортирует по (AI-score desc, TF-IDF desc). Кандидаты всегда показываются.
   - Если AI считает что точного аналога нет — рядом висит предупреждение «⚠ точного аналога нет», но варианты для клика всё равно доступны.

4. **Обучение из примеров** (всегда работает)
   - Таблица `examples` в SQLite копит пары `(query_text, item_id, source)`.
   - На каждом поиске считается сходство текущего запроса с прошлыми примерами (CHAR-ngram + точное совпадение query_norm). Похожий → boost его `item_id` в общем ранжировании. В UI помечается ★.
   - Источники: `click` (клик по строке таблицы), `file:*` (массовая загрузка из xlsx через `ingest_xlsx.py`).
   - Endpoints: `POST /api/feedback {query, item_id}` и `POST /api/ingest {examples:[{query,code|item_id}]}`.

В UI: две независимые галки «AI-расшифровка» и «AI-фильтр результатов»; любая строка результата кликабельна — клик записывается как обучающий пример.

## Ключи API

Скрипт ищет `GROQ_API_KEY` и `GEMINI_API_KEY` в `os.environ` и в `.env` (путь задан в `ENV_FILE` в `app.py`). Если оба отсутствуют — AI-стадия отключается, остаётся обычный TF-IDF поиск + обучение по примерам. Если есть только один — работает только он.

## Установка и запуск

```powershell
pip install -r requirements.txt

# 1. Собрать индекс (~1–2 мин на 175k строк)
python build_index.py "путь\к\Классификатору.xlsx"

# 2. (опционально) Загрузить заранее заполненные пары "запрос → код КСР" из xlsx
python ingest_xlsx.py "путь\к\заполненной_смете.xlsx"

# 3. Поднять веб-приложение
python app.py
```

Открыть http://127.0.0.1:5000/
